# 皮肤插件用法

## 命令（需要 OP / 管理员）

| 命令 | 说明 |
| --- | --- |
| `/skin set <玩家> <皮肤名>` | 给玩家套上皮肤，广播给所有人并记下来 |
| `/skin clear <玩家>` | 清掉记录并还原皮肤（读内存真实皮肤即时还原原皮，失败则套默认皮/等重进，见下文） |
| `/skin list` | 列出 `skins` 目录里能用的皮肤 |
| `/skin reload` | 重新扫描 `skins` 目录并把已设置的皮肤重发一遍 |
| `/skin show <皮肤名>` | 在自己面前临时生成一个假人预览皮肤（约 20 秒后消失） |

`/skin set` 的 `<皮肤名>` 也可以直接写一个 png 图片的 **URL**（http/https）：

```
/skin set Steve https://example.com/abc.png
```

插件会用 `curl` 把图下到 `skins/.urlcache/` 再套上（只支持 2D，没有自定义模型）。

## 前置依赖

- **GMLIB-LegacyRemoteCallApi**（必须）：换肤用 GMLIB 的二进制流发包，包头按版本对应，跨版本更稳。
- **iListenAttentively-LseExport**（`/clear` 即时还原原皮需要）：用它读内存里的真实皮肤。
  要求其 `getString` 支持 base64 参数（用于二进制无损读取）；没装或版本旧会自动回退（见下文）。
- 系统自带 **curl**（Win10+ / Linux 都有）：URL 皮肤需要。

## 原理

换肤**发包**实现：设置皮肤时先给在线玩家发 `PlayerSkin` 包立即刷新，再改一遍玩家列表
（`PlayerList`）；新玩家进服时会自动从玩家列表拿到正确皮肤。设置过的皮肤存进 `players.json`，玩家退了再进会自动补上。
（`/skin clear` 还原原皮时会用 ila 读一次内存里的真实皮肤，见下文。）

`show` 命令发 `AddPlayer` 包，造一个只有你自己看得到的假人来预览皮肤，不占服务器实体。

### 关于 clear

发包换肤只改客户端、不动服务器内存，所以服务器里一直存着玩家的「原皮」。`clear` 按下面顺序还原：

1. **读内存真实皮肤即时还原**（需 iListenAttentively-LseExport，默认开启）：用符号调用
   `getLevel → getPlayerList` 定位玩家，`SerializedSkinImpl::write` 把原皮序列化，再用 `getString(addr, true)`
   以 base64 **无损**读出字节、发包还原，**瞬间变回原皮**（含 Persona 市场皮肤），无需重进、也不用提前存盘；
2. 读内存失败（没装 ila / 旧版 `getString` 无 base64 参数 / 符号对不上）时：有 `skins/default.png`
   （或 `skins/default/` 4D 文件夹）就发包套默认皮，立即生效；
3. 再不行就只删记录，玩家**重新进服**后服务器把真实皮肤重新广播，自然恢复。

> 实现说明：内存读用的是稳定导出符号 + 标准库 ABI 偏移，失败会自动回退到第 2/3 步、不会崩服。
> 关掉这条可在 `skin.js` 把 `ENABLE_MEM_RESTORE` 设为 `false`。
> Persona/4D 皮肤序列化后可达数百 KB，`clear` 发包时可能卡顿一两秒，属正常。

### URL 皮肤

`set` 的皮肤名传 http/https 的 png 链接时，插件用系统的 `curl` 下到 `skins/.urlcache/`
（文件名按链接算 hash，同一链接复用）。下好后跟普通皮肤一样存进 `players.json`，玩家退进会自动补；
缓存还在就不重复下。

> 注意：URL 只支持 2D 普通皮肤，不带自定义模型；换了同一个链接背后的图想刷新，
> 把 `skins/.urlcache/` 里对应文件删了再 `set` 一次。

## 皮肤怎么放

皮肤都放在 `plugins/skin/skins/` 目录里，两种摆法：

### 1）普通皮肤（2D）

直接丢一张 png 进去：

```
skins/小明.png
```

皮肤名就是文件名（不带 `.png`），比如上面这个皮肤名是 `小明`。贴图按标准人物模型走（64x64 或 64x32 都行）。

### 2）4D 皮肤（带自定义模型）

建一个文件夹，名字就是皮肤名，里面放一张贴图 png + 一个 geometry.json：

```
skins/龙娘/            <- 皮肤名是 "龙娘"
  texture.png          <- 贴图（名字随意，认 .png）
  geometry.json        <- 模型文件（名字随意，认 .json）
```

png 和 json 各放一个就够了，插件会自动认。geometry.json 是基岩版的几何模型文件，新老格式都支持：

- 新格式：里面有 `minecraft:geometry`，会自动读 `description.identifier`
- 老格式：键名形如 `geometry.xxx`，会自动识别

模型和贴图都是直接打进数据包发给客户端的，客户端不需要再装资源包。

> 注意：4D 皮肤的贴图尺寸要跟模型对得上（模型里 `texture_width` / `texture_height` 是多少，png 就得是多大），不然会显示错乱。

## 给别的插件用的接口（`ll.exports`，命名空间 `SkinAPI`）

别的插件用 `ll.imports("SkinAPI", "函数名")` 拿到函数再调。

| 函数 | 返回 | 说明 |
| --- | --- | --- |
| `setSkin(player, skinName)` | bool | 套皮肤并记录（玩家退进自动补） |
| `applySkin(player, skinName)` | bool | 只发包套皮肤，不写记录（临时） |
| `clearSkin(player)` | bool | 清掉记录并还原原皮（读内存真实皮肤即时还原，失败则默认皮/重进） |
| `showSkin(player, skinName)` | bool | 在该玩家面前展示假人皮肤（只他自己可见） |
| `listSkins()` | 数组 | 列出可用皮肤名 |
| `reload()` | bool | 重新扫描皮肤目录 |

`skinName` 就是 `skins/` 目录里的皮肤名（png 文件名或 4D 文件夹名）。

例子：

```js
const setSkin = ll.imports("SkinAPI", "setSkin");
setSkin(player, "龙娘");
```
